# OGSM 專案 - Claude Code 操作說明

## 檔案分工
- `index.html`：HTML 結構，禁止在此寫 CSS 或 JS
- `style.css`：所有樣式，修改樣式只動此檔
- `script.js`：所有邏輯，修改功能只動此檔
- `Code.gs`：Google Apps Script 後端邏輯

## 修改規則
- 改樣式（顏色、排版、動畫）→ 只修改 style.css
- 改功能（邏輯、計算、互動）→ 只修改 script.js
- 改頁面結構（新增區塊、調整 HTML）→ 只修改 index.html
- 每次任務請只讀取並修改對應的單一檔案

## 專案功能說明

OGSM 策略看板系統，串接 Google Sheets（透過 Google Apps Script）作為資料後端，提供以下功能：

- **三欄看板**：G（支線目標）、S（策略）、M（行動項目）三欄聯動展示，點擊目標可篩選策略與行動
- **燈號系統**：每個支線目標有紅/黃/綠三色燈號，可點擊切換並自訂各燈號意涵（儲存於 localStorage）
- **策略自動進度計算**：策略進度依「完成」狀態的行動數量自動計算百分比
- **目的整體進度**：頂部顯示所有支線目標進度的平均值
- **行動狀態管理**：行動項目支援「未開始／進行中／完成／卡關」四種狀態，點擊 badge 可快速切換
- **行內編輯**：目的名稱、目標名稱、策略名稱、行動名稱皆可點擊直接編輯，失焦後自動同步至 Google Sheets
- **新增功能**：支援新增支線目標（含顏色選擇）及新增行動項目（含策略名稱、負責人、截止日期、進度、狀態）
- **Toast 通知**：操作成功或失敗皆有底部提示訊息
- **Modal 互動**：新增/編輯操作以 modal 彈窗呈現，支援 Escape 鍵與點擊背景關閉

## 預設行為規則
收到修改需求時，請先判斷影響範圍，
只讀取並修改最少量的必要檔案，
不得同時開啟超過一個檔案，除非需求明確跨檔。
